# Agent Performance Metrics
> **Purpose:** Track each agent's efficiency to identify bottlenecks and weak links in the pipeline.
> **Maintained By:** Task Orchestrator (Updated after every task completion or Feedback Loop cycle).

## How To Read This File
- **Retry Count** = How many times the Feedback Loop sent work back to this agent.
- **High retry count** on Coder = Architect's plans may be unclear.
- **High retry count** on Reviewer = Coding standards may need better definition.
